Overview

Templates are page-level HTML structures that define the layout and section composition for a specific type of document. Each template combines the shell layout, header, hero, main content area, sidebar (where applicable), and footer into a complete, ready-to-use page structure. Four templates are available: Documentation, Dashboard, Diagram, and Reference. All templates are located in src/templates/.

Documentation Template

Standard template for technical documentation pages. Includes a sidebar with table of contents navigation and a metadata panel.

Feature Value
Layout Two-column with sidebar
Sidebar TOC navigation + metadata panel
Use Case Specs, guides, reference docs, API documentation
Status Stable

Dashboard Template

Overview template for summarized project or system state. Full-width layout with metric widgets and status cards. No sidebar.

Feature Value
Layout Full-width, single column
Sidebar None
Use Case Project dashboards, system overviews, status hubs
Status Stable

Diagram Template

Full-page template for complex visualizations and architectural diagrams. Full-width layout centered on the diagram panel. No sidebar. Loads diagrams.css.

Feature Value
Layout Full-width, diagram-centered
Sidebar None
Use Case Architecture diagrams, roadmaps, flow charts
Status Stable

Reference Template

Lookup and catalog template for glossaries, component references, and token catalogs. Uses a sidebar for alphabetical or category-based index navigation.

Feature Value
Layout Two-column with sidebar
Sidebar Alphabetical or category index
Use Case Glossaries, token catalogs, component references
Status Stable

Notes

All templates use the same CSS import stack. The Diagram template additionally loads diagrams.css for the diagram framework classes. Templates are starter files — copy the relevant template and replace placeholder content to create a new page in the system.
