Overview

The AI-SDOS Documentation Design System (DDS) is a static, framework-free design system built entirely with HTML, CSS, and inline SVG. It provides a consistent visual language for producing technical documentation, architectural diagrams, and reference catalogs without requiring JavaScript, build tools, or external dependencies. Every page in the system works standalone by opening the HTML file directly in a browser.

Principles

Four non-negotiable constraints shape every decision in the system:

  • Semantic Tokens Only — Components reference semantic CSS custom properties (e.g., --text-primary), never raw primitive values. This ensures consistent theming and future-proofs the system against token value changes.
  • No JavaScript — All interactivity is achieved through CSS and HTML alone. No scripts, no event listeners, no dynamic state. Pages are fully static.
  • No Frameworks — Zero runtime dependencies. No React, Vue, or any CSS framework. The CSS layer is authored entirely in-house and is fully self-contained.
  • Print-ready — All pages include a dedicated print.css layer that produces clean, formatted output when printed or exported via the browser print dialog.

Architecture

The CSS stack is organized into ordered layers. Each layer builds on the previous and must be loaded in the sequence below.

Layer File Purpose
1 — Reset reset.css Normalizes browser defaults and establishes a clean baseline.
2 — Tokens tokens.css Defines all CSS custom properties (primitive and semantic).
3 — Typography typography.css Font stack, scale, line-height, and heading styles.
4 — Layout layout.css Shell, header, hero, main, sidebar, and footer structures.

Conventions

Naming and structural conventions applied consistently across the system:

  • BEM-style class naming — dds-block__element--modifier
  • All classes prefixed with dds- — prevents collisions with third-party CSS
  • Utility classes are unprefixed — e.g., mt-md, text-secondary, grid-cols-3
  • No inline styles — except for absolute-positioned SVG diagram nodes (position:absolute; top:Xpx; left:Xpx)
  • Semantic HTML elements — header, main, aside, nav, footer, section, article

Notes

The Foundation layer is stable and unlikely to change. Any modifications to reset, tokens, or the CSS layer order will propagate across the entire system and require full regression testing across all templates and diagram types.
