Overview

The Component Library provides the base UI building blocks used across all DDS pages. Components are implemented as pure HTML elements with BEM-style CSS classes. They reference semantic tokens exclusively and contain no JavaScript. Six components are currently stable: Badge, Card, Divider, Legend, Header, and Footer.

Badges

Badges communicate status, category, or metadata. Five semantic variants are available.

Badge Primary

Used for active state, version labels, and primary category.

Primary

Badge Success

Used for complete status, stable state, and positive outcomes.

Success

Badge Neutral

Used for pending status, draft state, and informational labels.

Neutral

Cards

Cards are the primary surface for grouping related content. They support optional header, body, and footer sub-elements.

Card with Header

A standard card with a header section. Use for titled content blocks, component showcases, or grouped information.

A card without a header. Use for notes, supplementary content, or any block that does not require a title.

Dividers

Dividers provide visual separation between sections and list items. They are implemented as hr elements with the dds-divider class.

Legend

Legends accompany diagrams to explain color and node variants. Each legend item pairs a color swatch with a descriptive label.

Primary — active state
Success — complete state
Neutral — default / pending

Notes

All components are stable and covered by the base CSS layers. Additional component variants (warning, error) are available for Badges and Status Cards. See the dashboard template for Status Card usage in context.
