Component

UI

A reusable HTML and CSS building block that encapsulates a specific UI pattern. Components in the DDS are implemented as pure HTML elements styled via BEM class names. They contain no JavaScript and reference only semantic tokens.

Related: Pattern, Template, Design Token

Design Token

Foundation

A named CSS custom property that stores a design decision — such as a color value, spacing unit, or font size. Tokens abstract raw values from their usage, enabling consistent theming and easier system-wide updates.

Related: Primitive Token, Semantic Token, tokens.css

DDS

Acronym

Documentation Design System. The full name of this project: a static, framework-free design system for producing technical documentation, architectural diagrams, and reference catalogs using HTML, CSS, and SVG only.

Related: Shell, Template, Foundation

Diagram Framework

Diagrams

The CSS and SVG system that provides classes and primitives for rendering technical diagrams. It includes node variants, group containers, connector styles, legend components, and the diagram panel layout. Implemented in diagrams.css.

Related: Component, DDS, Template

Export System

Planned

Planned tooling (Phase 3) to output DDS pages as PDF files and diagrams as standalone SVG assets. Not yet implemented. Manual browser print-to-PDF is available as an interim solution via the included print.css layer.

Related: Foundation, Template

Foundation

Layer

The base layer of the DDS CSS stack. Comprises the reset, design tokens, and typography files. Every other CSS layer and component depends on the Foundation being loaded first.

Related: Design Token, DDS, Typography

Pattern

UI

A reusable composition of components that solves a recurring layout or content problem. Examples in the DDS include the Hero pattern, Metadata Panel pattern, and Section Header pattern. Patterns are documented in src/patterns/.

Related: Component, Template

Primitive Token

Foundation

A raw design value stored as a CSS custom property — for example, a specific hex color code or a pixel measurement. Primitive tokens are the source of truth for all visual values but must never be referenced directly in component styles. Use semantic tokens instead.

Related: Semantic Token, Design Token

Semantic Token

Foundation

A token that expresses design intent rather than a raw value. For example, --text-primary communicates the role (primary text color) while its actual value references a primitive token. Components must only use semantic tokens.

Related: Primitive Token, Design Token, Component

Shell

Layout

The dds-shell element is the outermost layout wrapper present on every DDS page. It defines the page structure using named slots for the header, hero, main content, and footer regions, and provides consistent vertical rhythm across all templates.

Related: Template, Foundation, Layout

Template

Structure

A page-level HTML structure designed for a specific document type. Templates in the DDS compose the shell, header, hero, main, and footer into a complete, ready-to-use page. Four templates are available: Documentation, Dashboard, Diagram, and Reference.

Related: Shell, Pattern, Component

Typography

Foundation

The type scale and text styling system for the DDS. Defined in typography.css, it establishes font family, heading hierarchy (h1–h4), body text, small text, and caption styles. All typography references semantic font tokens.

Related: Foundation, Design Token, Semantic Token

Notes

This glossary covers the core vocabulary of the AI-SDOS Documentation Design System. Terms are organized alphabetically. Additional entries will be added as the system evolves toward v1.0.
