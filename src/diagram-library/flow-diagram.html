Flow Diagram

Shows the sequential steps of a process or data flow — from a start event through processing stages to a final outcome.

Behavioral

Process Step
Decision / Validation
Terminal (Start / End)

Arrows indicate the direction of execution. Decision nodes can branch; this example shows a happy-path linear flow.

Notes

A Flow Diagram (or Flowchart) is the most general diagram type for documenting processes. Use it to show both happy paths and error branches. Keep each box to a single action or decision to maintain clarity. When the diagram becomes complex, consider splitting it into sub-flows.
