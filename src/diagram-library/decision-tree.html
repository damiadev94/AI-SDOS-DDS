Branch labels (Yes / No) indicate the condition under which each path is taken. Every leaf node is a terminal outcome.

Notes

Decision Trees make conditional logic explicit and auditable. They are particularly useful for modelling access control policies, pricing rules, and routing logic. Each decision node should have exactly two branches (or a clearly defined set of mutually exclusive conditions).
