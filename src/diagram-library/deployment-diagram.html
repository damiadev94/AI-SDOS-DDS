Deployment environments (cloud, edge) are shown as dashed groups. Each node represents a physical or virtual infrastructure element.

Notes

A Deployment Diagram maps the software containers from the Container Diagram onto the infrastructure — cloud regions, servers, virtual machines, containers, or managed services. It is useful for DevOps planning, capacity estimation, and understanding failure domains.
