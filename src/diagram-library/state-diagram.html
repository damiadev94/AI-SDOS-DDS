State Diagram

Shows all possible states of an entity and the transitions — triggered by events or conditions — that move it between those states.

Transitions are labelled with the event or guard that triggers them. Every state machine must have an initial state and at least one terminal state.

Notes

State Diagrams are ideal for modelling entities with well-defined lifecycles: orders, documents, tickets, subscriptions. They make invalid transitions explicit, which helps teams implement correct status validation logic in code and database constraints.
