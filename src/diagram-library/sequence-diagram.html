Messages are numbered in chronological order. Outgoing calls are shown with neutral badges; return messages use success badges.

Notes

Sequence Diagrams are the standard way to document API contracts, integration flows, and authentication protocols. They capture the exact order of messages and which participant initiates each call. Model one scenario (happy path) per diagram; document error flows separately.
