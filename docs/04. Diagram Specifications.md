# AI-SDOS Documentation Design System (DDS)

# 04. Diagram Specifications

---

# Objetivo

Definir la arquitectura, clasificación y especificación de todos los tipos de diagramas soportados por el Documentation Design System.

Cada tipo de diagrama representa una forma estandarizada de comunicar información y posee una estructura, propósito y conjunto de componentes claramente definidos.

Los diagramas forman parte del lenguaje visual del DDS y constituyen la capa de comunicación entre la documentación y el lector.

---

# ¿Qué es un Diagram?

Un Diagram es una representación visual estructurada construida mediante la composición de componentes reutilizables.

Todo diagrama:

- comunica una única idea principal;
- posee una estructura consistente;
- utiliza únicamente componentes del DDS;
- reutiliza Design Tokens;
- es independiente del contenido específico;
- puede exportarse sin modificaciones.

---

# Principios

Todo diagrama debe cumplir los siguientes principios.

- Single Purpose
- Information First
- Component Based
- Token Driven
- Consistent
- Readable
- Tool Agnostic
- Printable
- Responsive
- Reusable

---

# Clasificación

## Project Diagrams

Representan el estado y evolución de un proyecto.

Ejemplos:

- Project State
- Roadmap
- Dashboard
- Timeline

---

## Conceptual Diagrams

Representan ideas, conceptos y relaciones semánticas.

Ejemplos:

- Mind Map
- Concept Map
- Knowledge Graph
- Domain Map

---

## Structural Diagrams

Representan la estructura de un sistema.

Ejemplos:

- UML Class Diagram
- C4 Context
- C4 Container
- ER Diagram
- Architecture Diagram

---

## Behavioral Diagrams

Representan comportamiento y flujo.

Ejemplos:

- Flow Diagram
- Sequence Diagram
- State Machine
- Decision Tree
- Activity Diagram

---

## Reference Diagrams

Representan información de consulta.

Ejemplos:

- Taxonomies
- Hierarchies
- Matrix
- Tables
- Legends

```
Design Tokens
        │
        ▼
Primitive Components
        │
        ▼
Layout Components
        │
        ▼
Diagram Components
        │
        ▼
Diagram
```

---

# Reglas

Todo diagrama deberá:

- representar una única idea principal;
- utilizar componentes reutilizables;
- evitar elementos decorativos innecesarios;
- mantener consistencia visual;
- respetar los Design Tokens;
- permitir impresión en A4;
- mantener legibilidad en formato digital.

---

# Evolución

Los diagramas podrán:

- incorporar nuevos componentes;
- incorporar nuevas variantes;
- redefinir layouts;
- evolucionar manteniendo compatibilidad.

Nunca deberán romper la identidad visual del DDS.
