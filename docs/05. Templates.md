# AI-SDOS Documentation Design System (DDS)

# 05. Templates

---

# Objetivo

Definir las plantillas reutilizables que estructuran los documentos del Documentation Design System.

Los Templates establecen la organización visual de una página, determinando cómo se distribuyen títulos, diagramas, tablas, componentes y contenido.

No contienen información específica del proyecto.

Representan únicamente la estructura.

---

# ¿Qué es un Template?

Un Template es una composición reutilizable de Layouts, Components y Patterns que define la estructura de un documento.

Todo Template:

- organiza contenido;
- reutiliza componentes existentes;
- no contiene lógica documental;
- no contiene información específica;
- mantiene una estructura consistente;
- puede reutilizarse en múltiples proyectos.

---

# Principios

Todo Template debe cumplir los siguientes principios.

- Structure First
- Composition over Duplication
- Responsive
- Printable
- Reusable
- Consistent
- Minimal
- Information First

```
        ▼
Pages
```

Los Templates representan el último nivel reutilizable antes de generar un documento.

---

# Template Anatomy

Todo Template podrá componerse de las siguientes zonas.

```
Header

↓

Overview

↓

Primary Content

↓

Supporting Content

↓

Notes

↓

Footer
```

Cada Template podrá utilizar únicamente las secciones que resulten necesarias.

---

# Reglas

Todo Template deberá:

- reutilizar únicamente componentes del DDS;
- respetar la jerarquía visual;
- utilizar los Design Tokens;
- mantener consistencia entre documentos;
- permitir impresión en A4;
- ser adaptable a formato web;
- evitar estructuras específicas de un único proyecto.

---

# Versionado

Todo Template podrá encontrarse en uno de los siguientes estados.

| Estado | Descripción |
|---------|-------------|
| Draft | En diseño |
| Stable | Aprobado |
| Deprecated | En proceso de reemplazo |
| Removed | Eliminado |

---

# Evolución

Los Templates podrán:

- incorporar nuevos Patterns;
- reorganizar Layouts;
- añadir variantes;
- adaptarse a nuevos formatos.

Nunca deberán romper la arquitectura del DDS.
