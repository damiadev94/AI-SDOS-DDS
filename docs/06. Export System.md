# AI-SDOS Documentation Design System (DDS)

# 06. Export System

---

# Objetivo

Definir la arquitectura, reglas y procesos que permiten transformar la documentación generada por el Documentation Design System en distintos formatos de salida, preservando la estructura, consistencia visual y calidad de presentación.

El Export System garantiza que un mismo documento pueda publicarse en múltiples formatos sin requerir modificaciones manuales.

---

# Propósito

Separar la generación del contenido de su representación final.

Los documentos deberán escribirse una única vez y exportarse a diferentes formatos utilizando un proceso estandarizado.

---

# Filosofía

El formato de salida nunca debe condicionar la construcción del documento.

Todo documento debe diseñarse una única vez.

El Export System será responsable de adaptar dicha representación al medio correspondiente.

---

# Arquitectura

```
Documentation

        │

        ▼

Templates

        │

        ▼

HTML

        │

        ▼

Export System

        │

────────┼───────────────

│       │        │

▼       ▼        ▼

PDF    SVG     PNG

                │

                ▼

           Print
```

HTML representa siempre la fuente principal.

Todos los demás formatos derivan de él.

---

# Principios

Todo proceso de exportación deberá cumplir los siguientes principios.

- Single Source of Truth
- HTML First
- Print First
- Lossless Export
- Consistent Output
- Reproducible
- Tool Agnostic
- Versionable

---

# Formatos soportados

## HTML

Formato principal del sistema.

Características:

- Editable
- Versionable
- Responsive
- Fuente oficial

---

## PDF

Formato destinado a documentación final.

Características:

- Vectorial
- Compatible con impresión
- Distribución
- Archivado

---

## SVG

Formato para diagramas individuales.

Características:

- Escalable
- Editable
- Vectorial
- Alta calidad

---

## PNG

Formato rasterizado.

Uso recomendado:

- Presentaciones
- Documentación externa
- Redes sociales
- Referencias rápidas

---

## Print

Salida optimizada para impresión física.

---

# Reglas

Todo documento deberá:

- exportarse sin modificaciones manuales;
- conservar la jerarquía visual;
- mantener la tipografía;
- respetar los Design Tokens;
- mantener la calidad vectorial cuando sea posible;
- preservar proporciones;
- evitar contenido dependiente del formato de salida.

# Compatibilidad

El Export System deberá funcionar correctamente en:

- Navegadores modernos
- Chrome
- Edge
- Safari
- Firefox

Y permitir generación mediante herramientas automatizadas.

---

# Automatización

El proceso de exportación deberá poder ejecutarse mediante scripts.

Ejemplos:

- Build
- Batch Export
- Continuous Documentation
- CI/CD

La exportación nunca deberá depender de acciones manuales repetitivas.

---

# Validación

Antes de considerar una exportación como válida deberán verificarse los siguientes criterios.

- Layout correcto.
- Sin desbordes.
- Sin pérdida de calidad.
- Tipografía consistente.
- Colores correctos.
- Diagramas completos.
- Márgenes correctos.
- Escalabilidad preservada.

---

# Versionado

Toda exportación deberá poder asociarse a:

- versión del documento;
- versión del DDS;
- fecha de generación.
