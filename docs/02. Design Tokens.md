# AI-SDOS Documentation Design System (DDS)

# 02. Design Tokens

---

# Objetivo

Definir el sistema de Design Tokens que actuará como la única fuente de verdad (Single Source of Truth) para todas las decisiones visuales del Documentation Design System.

Los Design Tokens representan valores abstractos reutilizables que describen la identidad visual del sistema sin depender de una implementación específica.

Todo componente, layout o diagrama deberá construirse utilizando exclusivamente estos tokens.

---

# ¿Qué es un Design Token?

Un Design Token es una unidad de diseño reutilizable que representa un valor visual del sistema.

Ejemplos:

- Un color.
- Un tamaño de fuente.
- Un espaciado.
- Un radio de borde.
- Una sombra.
- Una duración de animación.

Los Tokens permiten mantener consistencia y facilitan la evolución del sistema modificando un único punto de configuración.

---

# Principios

Los Design Tokens deben cumplir los siguientes principios:

- Single Source of Truth.
- Independientes de componentes.
- Independientes de diagramas.
- Independientes de implementación.
- Semánticos antes que descriptivos.
- Reutilizables.
- Versionables.
- Escalables.

---

# Jerarquía

Los Tokens se organizan en tres niveles.

```
Primitive Tokens
        │
        ▼
Semantic Tokens
        │
        ▼
Component Tokens
```

---

## Primitive Tokens

Representan valores puros.

Ejemplos:

- #2563EB
- 16px
- 8px
- 600

Nunca deberían utilizarse directamente por los componentes.

---

## Semantic Tokens

Asignan significado.

Ejemplo:

```
Color.Primary

Color.Success

Text.Primary

Border.Default

Spacing.Large
```

Los componentes utilizan únicamente Semantic Tokens.

---

## Component Tokens

Representan configuraciones específicas para componentes concretos.

Ejemplo:

```
Card.Background

Card.Padding

Node.Border

Badge.Success
```

---

---

# Reglas

Todo componente deberá:

- Utilizar únicamente Semantic Tokens.
- No utilizar colores hardcodeados.
- No utilizar tamaños arbitrarios.
- No utilizar espaciados fuera de la escala definida.
- No definir tipografías propias.
- No modificar Primitive Tokens.

---

# Evolución

Los Tokens pueden:

- ampliarse;
- versionarse;
- reorganizarse.

Nunca deberán eliminarse sin un proceso de deprecación.
